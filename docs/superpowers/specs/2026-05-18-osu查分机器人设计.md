# osu! 查分机器人设计文档

## 项目概述

**名称**: osubot-rs
**功能**: 兼容 OneBot 11 协议的 osu! 玩家数据查询机器人
**语言**: Rust (stable)
**协议**: OneBot 11 正向 WebSocket

## 核心功能

| 命令 | 说明 |
|------|------|
| `~ [模式]` | 查询自己（需绑定） |
| `~1` / `~,1` | 查询 taiko |
| `~2` / `~,2` | 查询 catch |
| `~3` / `~,3` | 查询 mania |
| `where <用户名> [, 模式]` | 查询他人 |
| `查@<QQ用户> [, 模式]` | 查询他人 |
| `绑定 <osu用户名>` | 绑定 osu! 账号 |
| `解绑` | 解除绑定 |

模式代码: `0`=std, `1`=taiko, `2`=catch, `3`=mania

## 技术栈

- **异步运行时**: Tokio 单线程事件循环
- **存储**: SQLite（用户绑定）
- **配置**: TOML 配置文件
- **API**: osu! API v2

## 模块设计

### osubot-core

**commands.rs** - 命令解析
- `Command` enum: 查询自己、查询他人、绑定、解绑
- `parse(raw: &str) -> Option<Command>`
- 解析 `~`、`where`、`查@`、`绑定`、`解绑`

**api.rs** - API 调用
- `fetch_user_stats(api_key: &str, username: &str, mode: GameMode) -> Result<UserStats>`
- 使用 reqwest 异步 HTTP 请求 osu! API v2
- 返回: pp, rank, country_rank, ranked_score, accuracy, playcount, hits, playtime, rank_change, country_rank_change

**storage.rs** - 存储
- `UserBinding { qq: i64, osu_username: String }`
- `bind(qq: i64, username: &str)`
- `unbind(qq: i64)`
- `get_binding(qq: i64) -> Option<String>`
- 使用 SQLite via rusqlite

**response.rs** - 格式化
- `format_user_stats(stats: &UserStats, mode: GameMode) -> String`
- 数值千位分隔、排名变化箭头、游玩时间格式化

### osubot

**config.rs** - 配置
```toml
[osu]
api_key = "your-api-key"

[bot]
onebot_url = "ws://127.0.0.1:8080"
```

**main.rs** - 主循环
- 连接 OneBot 11 WebSocket
- 使用 `tokio::sync::mpsc` 接收消息
- `select!` 轮询处理命令
- 快速回复"正在处理"后 spawn_blocking 处理耗时任务
- 主线程继续处理其他 WebSocket 消息

## 异步处理规范

### CPU 密集型任务（必须使用 spawn_blocking）

所有耗时的同步操作必须通过 `tokio::task::spawn_blocking` 执行：
- 图片生成
- 图像处理
- 文件压缩
- 任何同步重计算

**示例代码**（命令处理预留）:
```rust
// 命令处理分支示例
match command {
    Command::GenerateImage(_) => {
        let sender = resp_tx.clone();
        // 快速回复
        let _ = sender.send("正在生成图片，请稍候...".to_string()).await;

        // spawn_blocking 处理耗时任务
        let result = tokio::task::spawn_blocking(|| {
            // 图片生成逻辑（同步代码）
            generate_image()
        }).await;

        match result {
            Ok(data) => {
                let _ = sender.send(data).await;
            }
            Err(_) => {
                let _ = sender.send("处理失败".to_string()).await;
            }
        }
    }
    _ => {}
}
```

### I/O 操作（直接使用 async）

以下操作直接使用异步函数，不需要 spawn_blocking：
- HTTP 请求（reqwest）
- 数据库查询
- 文件读写

### 状态共享

spawn_blocking 内访问共享状态必须使用 `Arc`：
```rust
let shared_state = Arc::new(Mutex::new(State::new()));
// 在 spawn_blocking 闭包中
let state = shared_state.lock().unwrap();
```

禁止使用 `Rc`/`RefCell`，因为 spawn_blocking 跨线程。

### 可选：并发限制

使用 `tokio::sync::Semaphore` 限制同时进行的重任务数量：
```rust
let semaphore = Arc::new(Semaphore::new(5)); // 最多 5 个并发
let _permit = semaphore.acquire().await;
```

## 响应格式

```
{username}的个人信息—{模式名}
{pp}pp 表现
#{rank} (↑{变化} 或 ↓{变化} 或 —)
China #{country_rank} (↑{变化} 或 ↓{变化} 或 —)
{Ranked谱面总分}m Ranked谱面总分
{acc}% 准确率
{playcount} 游玩次数
{hit次数} 总命中次数
{playtime} 游玩时间
```

数值使用千位分隔符（`,`），排名变化：无变化显示 `—`。

## 行为规则

1. 用户未绑定时查询自己 → 提示"请先绑定 osu! 用户名"
2. API 请求失败 → 回复"查询失败，请稍后重试"
3. 用户不存在 → 回复"未找到该用户"
4. 命令格式错误 → 回复使用说明
